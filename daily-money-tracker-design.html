Implementation Plan

3-Sprint Build Roadmap

Six weeks of focused PDI work, from data model to portal UI to automation flows. Each sprint delivers a testable, usable milestone.

Sprint 1 · Weeks 1–2 Foundation

Data Model & Scoped App

  • Create scoped app x_pfmt in Studio
  • Build all 6 custom tables with field specs
  • Configure ACLs (user_profile row-level)
  • Seed system-default categories (15 rows)
  • Basic Business Rule: balance update on txn insert
  • ATF test: create account + transaction
✅ Deliverable: Working data layer on PDI
Sprint 2 · Weeks 3–4 UI & Automation

Portal + Flow Designer

  • Service Portal: 5 pages (dashboard, add txn, budgets, goals, analytics)
  • Angular widgets: balance card, transaction list, budget bars
  • Client Scripts: auto-date, category suggest, amount format
  • Flow 1 (Add Transaction) + Flow 4 (Recurring)
  • Notifications: budget alert email template
  • Now Mobile compatible page layout
✅ Deliverable: Functional portal with real data
Sprint 3 · Weeks 5–6 Analytics & Polish

Analytics, Goals & Handover

  • Platform Analytics KPIs: savings rate, avg daily spend
  • Flow 2 (Monthly Reset) + Flow 3 (Goal Progress)
  • Scripted REST API for external data import
  • Now Assist prompt integration (spend analysis)
  • Full ATF test suite across all flows
  • Export Update Set for backup
✅ Deliverable: Production-ready PFMT app

⚠️ PDI Build Reminders

  • Always scope your app — never build in Global
  • Use x_pfmt_ prefix on all table/field names
  • Test Business Rules on After insert, not Before, for balance updates
  • Use GlideAggregate for spent_amount SUM queries
  • Export Update Set after every Sprint milestone
  • Enable ATF in PDI settings before Sprint 1
