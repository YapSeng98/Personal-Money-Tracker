Before you start: Log into your PDI at dev275144.service-now.com and confirm Flow Designer and Service Portal plugins are active. Go to System Definition → Plugins to check.
1
Create the Scoped Application
Studio · ~10 minutes
1

Open Studio

In your PDI, click the main menu (top-left grid icon). Type Studio in the filter, then click it to open the App IDE in a new tab.

All→ System Applications→ Studio
Open Studio ↗
2

Create Application

In Studio, click Create Application. Fill in the fields exactly as shown:

Name : Personal Finance Money Tracker Scope : x_pfmt Version : 1.0.0 Short desc : Daily money tracking and budgeting app

Click Create. Studio will reload inside your new scoped app.

3

Confirm Scope is Active

In the top-right corner of the ServiceNow banner, you should see the scope switcher showing Personal Finance Money Tracker. If it still says Global, click the scope picker and switch to your new app. All work from here must be done inside this scope.

🚫 Never create PFMT tables or scripts in the Global scope — they won't be packaged with your app and will break on export.
2
Build the 6 Custom Tables
Studio → Create File → Table · ~30 minutes
ℹ️ For each table: in Studio click Create File → Data Model → Table. Set the exact name below — ServiceNow will auto-prefix it with x_pfmt_.
1

Create Table: x_pfmt_user_profile

Table name : user_profile → becomes x_pfmt_user_profile Label : User Profile Extends : Task [task] → or leave blank Add fields : user_name String(100) Mandatory email Email currency_preference String(3) Default: SGD monthly_income_target Currency
2

Create Table: x_pfmt_account

Table name : account Add fields : user_profile Reference → x_pfmt_user_profile Mandatory account_name String(100) Mandatory account_type Choice [checking,savings,credit_card,cash,investment] institution_name String(100) current_balance Currency Default: 0 currency String(3) Default: SGD is_active True/False Default: true
3

Create Table: x_pfmt_category

Table name : category Add fields : user_profile Reference → x_pfmt_user_profile category_name String(80) Mandatory category_type Choice [expense, income] icon_emoji String(10) color_hex String(7) is_system_default True/False Default: false
4

Create Table: x_pfmt_transaction ← Core table

Table name : transaction Add fields : account Reference → x_pfmt_account Mandatory category Reference → x_pfmt_category Mandatory transaction_type Choice [expense, income, transfer] Mandatory amount Currency Mandatory description String(255) Mandatory transaction_date Date Mandatory state Choice [1=Draft, 2=Confirmed, 3=Voided] Default: 2 is_recurring True/False Default: false recurring_frequency Choice [daily, weekly, monthly] next_run_date Date notes String(1000) attachment Attachment
5

Create Table: x_pfmt_budget

Table name : budget Add fields : user_profile Reference → x_pfmt_user_profile Mandatory category Reference → x_pfmt_category Mandatory budget_name String(100) period_type Choice [weekly, monthly, yearly] Default: monthly budget_amount Currency Mandatory spent_amount Currency Default: 0 remaining_amount Currency Default: 0 alert_threshold_pct Integer Default: 80 period_start Date period_end Date rollover_enabled True/False Default: false
6

Create Table: x_pfmt_savings_goal

Table name : savings_goal Add fields : user_profile Reference → x_pfmt_user_profile Mandatory account Reference → x_pfmt_account goal_name String(100) Mandatory target_amount Currency Mandatory current_amount Currency Default: 0 monthly_contribution Currency target_date Date goal_status Choice [in_progress, achieved, paused] Default: in_progress icon_emoji String(10)
7

Seed Default Categories

Navigate to Scripts - Background. Open GR_Utilities.js from your PFMT folder, copy the seedCategories() function body, paste it, and click Run script. This creates all 13 default categories.

All→ System Definition→ Scripts - Background
Open Scripts - Background ↗
3
Deploy Scripts from Your Code Files
Studio · Business Rules · Client Scripts · Script Includes · ~40 minutes

For each script below: in Studio click Create File, choose the type, fill in the settings, and paste the matching file from your PFMT folder.

Business Rules — Studio → Create File → Server Development → Business Rule

File to PasteTableWhenOperationCondition
BR_ValidateTransaction.js x_pfmt_transaction Before Insert + Update none
BR_UpdateAccountBalance.js x_pfmt_transaction After Insert + Update state = 2
BR_UpdateBudgetSpent.js x_pfmt_transaction After Insert transaction_type = expense
BR_BudgetCalculatedFields.js x_pfmt_budget Before Insert + Update none

Script Include — Studio → Create File → Server Development → Script Include

1

SI_PFMTBudgetHelper.js

Name: PFMTBudgetHelper — tick the "Client callable" checkbox — paste the file content.

⚠️ If "Client callable" is not ticked, the GlideAjax call from the Client Script will silently fail.

Client Scripts — Studio → Create File → Client Development → Client Script

File to PasteTableTypeField
CS_DefaultTransactionDate.js x_pfmt_transaction onLoad —
CS_FilterCategoriesByType.js x_pfmt_transaction onChange transaction_type
CS_RecurringToggleAndBudgetHint.js x_pfmt_transaction onChange is_recurring, category

Flow Designer — Open Flow Designer → New Flow

1

Flow 2 — Monthly Budget Reset

In Flow Designer: New → Flow → Name it PFMT Monthly Budget Reset → Add trigger: Date · Scheduled Daily → set to 1st of month 00:01 → Add action: Script → paste FLOW_MonthlyBudgetReset.js → Activate.

Open Flow Designer ↗
2

Flow 4 — Recurring Transactions

New Flow → Name: PFMT Recurring Transactions → Trigger: Date · Scheduled Daily at 08:00 → Add action: Script → paste FLOW_RecurringTransactions.js → Activate.

Scripted REST API — All → System Web Services → Scripted REST APIs

1

Create the PFMT REST API

Name : PFMT API API ID : pfmt Base path : /pfmt/v1

Inside the new API, click Resources → New:

Name : Transactions Path : /transactions Methods : GET, POST → Paste REST_TransactionsAPI.js into the Script field

Your live endpoint will be:
https://dev275144.service-now.com/api/x_pfmt/pfmt/v1/transactions

4
Build the Service Portal Widget
Service Portal Designer · ~20 minutes
1

Open Service Portal

All→ Service Portal→ Service Portal Configuration
Open Portal Designer ↗
2

Create New Widget

In the Portal Config sidebar, click Widgets → Create a new widget. Name it PFMT Dashboard. Then paste each file into the matching tab:

FilePaste into Tab
PORTAL_Widget_HTML.htmlHTML Template
PORTAL_Widget_CSS.scssCSS
PORTAL_Widget_ClientController.jsClient Controller
PORTAL_Widget_ServerScript.jsServer Script

Click Save.

3

Add Widget to a Portal Page

In Service Portal Config → Pages → New → ID: pfmt_dashboard → Open in Designer → drag your PFMT Dashboard widget from the widget list onto the canvas → Save.

Your portal page will be live at:
https://dev275144.service-now.com/sp?id=pfmt_dashboard

5
Test, Verify & Go Live
ATF · Manual testing · Update Set export · ~20 minutes
1

Create a Test Transaction Manually

Navigate to your x_pfmt_transaction table list and click New. Fill in all mandatory fields and save. Then check:

  • Account current_balance decreased (for expense)
  • Matching Budget spent_amount increased
  • Transaction state = Confirmed
  • No error messages in the Activity log
Open x_pfmt_transaction ↗
2

Test the REST API with a GET Request

Open your browser and visit this URL while logged into your PDI. You should get a JSON response with your transactions:

GET https://dev275144.service-now.com/api/x_pfmt/pfmt/v1/transactions Expected response: { "result": [...], "count": 1 }

Or use the REST API Explorer in your PDI:

Open REST API Explorer ↗
3

Run ATF Tests

Navigate to Automated Test Framework → Tests → New. Create a test that inserts an x_pfmt_transaction record and asserts account.current_balance changed correctly.

Open ATF Tests ↗
4

Export an Update Set

Always export your work after each sprint. Navigate to Local Update Sets → New → name it PFMT Sprint 1 → set it as Current → do your work → then Complete → Export to XML. Keep this file as your backup.

Open Update Sets ↗
5

View Your Live Portal

Open the portal page in your browser. Log in if prompted. Your PFMT dashboard widget should show the balance card and recent transactions pulled live from your PDI tables.

Open Live Portal ↗
